今天遇到场景，导出年级报告（pdf），一个页面需要中间层（node）向底层请求50+数量接口提供数据，因为接口数量过多，数据请求完成还后需要生成pdf。不管怎么请求都会超时，用户端都会超时，平均一个报告至少需要1分钟
甚至更长。

技术方案：

  • Object.defineProperty set 避免深层回调
  • res.write(progress);
